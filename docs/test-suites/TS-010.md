## Test Cases

### TC-010-001: Dashboard is the default landing page after login
- **Requirement**: REQ-055
- **Type**: Route / Integration
- **Steps**:
  1. POST to `/login` with valid credentials
  2. Follow redirect
- **Expected**: Response redirects to `/` (dashboard), HTTP 302 → 200

### TC-010-002: Dashboard returns 200 for authenticated users
- **Requirement**: REQ-055
- **Type**: Route
- **Steps**:
  1. Log in as any authenticated user
  2. GET `/`
- **Expected**: HTTP 200 with "Dashboard" in body

### TC-010-003: Dashboard displays summary cards for Products, Repositories, Templates, Teams, Users
- **Requirement**: REQ-057
- **Type**: Route / Integration
- **Steps**:
  1. Log in as admin
  2. Create some test entities (product, team, repository, template, user)
  3. GET `/`
- **Expected**: Page contains stat cards with labels "Products", "Repositories", "Templates", "Teams", "Users"

### TC-010-004: Dashboard summary card counts reflect only active (non-archived) entities
- **Requirement**: REQ-057
- **Type**: Service / Integration
- **Steps**:
  1. Create 3 active products and 1 archived product
  2. Call `dashboard_service.get_dashboard_counts()`
- **Expected**: `counts['products']` == 3 (excludes archived)

### TC-010-005: Dashboard count is 0 when no active entities exist
- **Requirement**: REQ-057
- **Type**: Service
- **Steps**:
  1. Database has no active entities
  2. Call `dashboard_service.get_dashboard_counts()`
- **Expected**: All count values are 0

### TC-010-006: Repository list supports filtering by Team

### TC-010-010: Repository list route accepts and applies filter query params
