## Test Cases

### TC-007-001 — Repository model fields present
**Requirement:** REQ-027
**Description:** Verify all required fields are present on the Repository model.
**Steps:**
1. Create a Repository instance with all fields
2. Assert name, url, description, team_id, template_id, is_active, is_archived, created_at, updated_at, version are all accessible
**Expected:** All fields present with correct types and defaults

---

### TC-007-002 — RepositoryArtifactValue model fields present
**Requirement:** REQ-027
**Description:** Verify RepositoryArtifactValue model has all typed columns.
**Steps:**
1. Create a RepositoryArtifactValue instance
2. Assert repository_id, template_artifact_id, value_text, value_number, value_boolean, value_list_option_id are accessible
**Expected:** All fields present, nullable except FK columns

---

### TC-007-003 — Create repository successfully
**Requirement:** REQ-027, REQ-050
**Description:** Create a repository with valid data including template and team.
**Steps:**
1. Create a team and an active template with one artifact
2. Call `create_repository(name, url, team_id, template_id, ...)`
3. Assert repository is saved with correct fields
**Expected:** Repository created; RepositoryArtifactValue rows created for each active artifact

---

### TC-007-004 — Repository URL uniqueness — create rejected
**Requirement:** REQ-028
**Description:** Creating a repository with a duplicate URL is rejected.
**Steps:**
1. Create repository with URL `https://github.com/org/repo`
2. Attempt to create another repository with the same URL
**Expected:** `RepositoryServiceError` raised with message about duplicate URL

---

### TC-007-005 — Repository URL uniqueness — includes archived
**Requirement:** REQ-028
**Description:** URL uniqueness check applies even to archived repositories.
**Steps:**
1. Create and archive a repository with URL `https://github.com/org/old`
2. Attempt to create a new repository with the same URL
**Expected:** `RepositoryServiceError` raised

---

### TC-007-006 — Repository URL uniqueness — edit rejected
**Requirement:** REQ-028
**Description:** Updating a repository to a URL already in use is rejected.
**Steps:**
1. Create two repositories with different URLs
2. Attempt to update one to the other's URL
**Expected:** `RepositoryServiceError` raised

---

### TC-007-007 — Template immutable after creation
**Requirement:** REQ-029
**Description:** Updating template_id on an existing repository has no effect (or is rejected).
**Steps:**
1. Create a repository with template A
2. Call `update_repository(...)` without changing template_id (service ignores it)
3. Verify template_id is unchanged
**Expected:** Repository template_id remains the same as at creation
