## Test Cases

### TC-001-001: App Factory Creates Application Successfully
- **Requirement**: REQ-066
- **Description**: Verify that `create_app()` returns a valid Flask application instance with correct configuration.
- **Expected Result**: App is created, config values are loaded, extensions are initialized.

### TC-001-002: Health Check Endpoint Returns 200
- **Requirement**: REQ-064
- **Description**: Verify the `/health` endpoint returns HTTP 200 with a JSON status.
- **Expected Result**: GET `/health` returns `{"status": "healthy"}` with status code 200.

### TC-001-003: Database Connection Works
- **Requirement**: REQ-063
- **Description**: Verify the application can connect to the database and execute a simple query.
- **Expected Result**: A simple SQL query executes without error.

### TC-001-004: Base Model Mixin Has Required Fields
- **Requirement**: REQ-046, REQ-069
- **Description**: Verify the base model mixin includes `id`, `is_active`, `is_archived`, `created_at`, `updated_at`, and `version` fields.
- **Expected Result**: All fields exist with correct types and defaults.

### TC-001-005: Soft Delete Defaults Are Correct
- **Requirement**: REQ-046
- **Description**: Verify that new model instances default to `is_active=True`, `is_archived=False`.
- **Expected Result**: Default values are applied correctly on new record creation.

### TC-001-006: Optimistic Locking Version Field
- **Requirement**: REQ-069
- **Description**: Verify the `version` field defaults to 1 and is present on base model instances.
- **Expected Result**: `version` starts at 1 for new records.

### TC-001-007: CSRF Protection Is Active
- **Requirement**: REQ-069
- **Description**: Verify that Flask-WTF CSRF protection is initialized and active.
- **Expected Result**: POST requests without CSRF token are rejected.

### TC-001-008: Base Template Renders With Expected Elements
- **Requirement**: REQ-066
- **Description**: Verify the base HTML layout includes Bootstrap 5, HTMX, sidebar navigation, and navbar.
- **Expected Result**: HTML response contains Bootstrap CSS, HTMX script, sidebar navigation links, and navbar.

### TC-001-009: Dashboard Route Returns 200
- **Requirement**: REQ-066
- **Description**: Verify the dashboard placeholder route is accessible.
- **Expected Result**: GET `/` returns HTTP 200 with rendered HTML.

### TC-001-010: Alembic Migration Infrastructure Exists
- **Requirement**: REQ-063
- **Description**: Verify that Alembic migrations directory and configuration exist.
- **Expected Result**: `migrations/` directory exists with `alembic.ini` or equivalent config.
