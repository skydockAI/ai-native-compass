## Test Cases

### TC-005-001: List teams — active only by default
**Requirement:** REQ-019
**Description:** GET /teams/ returns only active (non-archived) teams by default.
**Steps:** Log in as any user, GET /teams/. Verify archived teams are not shown.
**Expected:** Response 200. Only active teams listed.

### TC-005-002: List teams — show archived
**Requirement:** REQ-019
**Description:** GET /teams/?archived=1 includes archived teams.
**Steps:** Log in as any user, GET /teams/?archived=1.
**Expected:** Response 200. Archived teams appear in list.

### TC-005-003: Admin can create a team
**Requirement:** REQ-019
**Description:** An Admin user can create a new team via POST /teams/new.
**Steps:** Log in as Admin, POST /teams/new with name and description.
**Expected:** Team created, redirect to list, success flash message.

### TC-005-004: Editor can create a team
**Requirement:** REQ-019
**Description:** An Editor user can create a new team.
**Steps:** Log in as Editor, POST /teams/new with valid data.
**Expected:** Team created successfully.

### TC-005-005: Viewer cannot create a team
**Requirement:** REQ-019
**Description:** Viewer access to create form is forbidden.
**Steps:** Log in as Viewer, GET /teams/new.
**Expected:** HTTP 403.

### TC-005-006: Admin can edit a team
**Requirement:** REQ-019
**Description:** An Admin user can update team name and description.
**Steps:** Log in as Admin, POST /teams/<id>/edit with new name.
**Expected:** Team updated, redirect to list, success flash.

### TC-005-007: Editor can edit a team
**Requirement:** REQ-019
**Description:** An Editor user can update a team.
**Steps:** Log in as Editor, POST /teams/<id>/edit with valid data.
**Expected:** Team updated successfully.

### TC-005-008: Viewer cannot edit a team
**Requirement:** REQ-019
**Description:** Viewer is forbidden from editing teams.
**Steps:** Log in as Viewer, GET /teams/<id>/edit.
**Expected:** HTTP 403.

### TC-005-009: Duplicate team name rejected on create
**Requirement:** REQ-020
**Description:** Creating a team with a name already used by another team raises an error.
**Steps:** Create team "Alpha". Attempt to create another team "Alpha".
**Expected:** UserServiceError raised with message indicating duplicate name. No second team created.

### TC-005-010: Duplicate team name rejected on update
**Requirement:** REQ-020
**Description:** Renaming a team to match an existing team's name is rejected.
**Steps:** Create teams "Alpha" and "Beta". Attempt to rename "Beta" to "Alpha".
**Expected:** TeamServiceError raised with clear error message.

### TC-005-011: Unique name allowed across archived teams
**Requirement:** REQ-020
**Description:** A name used by an archived team can still be reused.
**Steps:** Archive team "Alpha". Create new team "Alpha".
**Expected:** Second team created successfully (names scoped to active teams at service level, but DB constraint is global — test documents this boundary).

### TC-005-012: Archive team with no active repositories succeeds
**Requirement:** REQ-022, REQ-049
**Description:** A team with no active repositories can be archived.
**Steps:** Create a team with no repository assignments. Call archive_team().
**Expected:** Team is archived (is_archived=True, is_active=False). Empty blocking list returned.

### TC-005-013: Archive team blocked by active repositories
**Requirement:** REQ-022, REQ-049
**Description:** A team with active repositories cannot be archived; blocking repos are returned.
**Steps:** (Future: when Repository model exists) Create team, assign active repo. Call archive_team().
**Expected:** TeamServiceError raised listing blocking repository names.

### TC-005-014: Archived team can be reactivated
**Requirement:** REQ-019
**Description:** An Admin/Editor can restore an archived team.
**Steps:** Archive a team. POST /teams/<id>/reactivate.
**Expected:** Team restored (is_archived=False, is_active=True).

### TC-005-015: Admin can archive a team
**Requirement:** REQ-019, REQ-022
**Description:** Admin can archive a team that has no blocking repos.
**Steps:** Log in as Admin, POST /teams/<id>/archive.
**Expected:** Team archived, redirect to list, success flash.

### TC-005-016: Viewer cannot archive a team
**Requirement:** REQ-019
**Description:** Viewer is forbidden from archiving teams.
**Steps:** Log in as Viewer, POST /teams/<id>/archive.
**Expected:** HTTP 403.

### TC-005-017: Optimistic locking prevents stale update
**Requirement:** REQ-019
**Description:** update_team with wrong version raises TeamServiceError.
**Steps:** Fetch team version. Attempt update with version+999.
**Expected:** TeamServiceError raised with concurrency message.

### TC-005-018: Unauthenticated access to team list redirects to login
**Requirement:** REQ-019
**Description:** Unauthenticated GET /teams/ redirects to login.
**Steps:** Without logging in, GET /teams/.
**Expected:** Redirect to /login.
