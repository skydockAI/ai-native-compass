## Test Cases

### TC-011-001: AuditLog model stores all required fields
**Requirement:** REQ-060
**Scenario:** A new AuditLog entry is created with all fields populated.
**Expected:** Entry is persisted with correct timestamp, user_id, entity_type, entity_id, action, before_value, after_value.

### TC-011-002: AuditLog allows null user_id (system-generated events)
**Requirement:** REQ-060
**Scenario:** An AuditLog entry is created with user_id=None.
**Expected:** Entry is persisted without error; user_id is null.

### TC-011-003: Login event is logged
**Requirement:** REQ-059
**Scenario:** A user logs in successfully.
**Expected:** AuditLog entry created with action='login', entity_type='session', entity_id=user.id.

### TC-011-004: Logout event is logged
**Requirement:** REQ-059
**Scenario:** A logged-in user visits the logout route.
**Expected:** AuditLog entry created with action='logout', entity_type='session', entity_id=user.id.

### TC-011-005: User create event is logged
**Requirement:** REQ-059
**Scenario:** Admin creates a new user via user_service.create_user().
**Expected:** AuditLog entry created with action='create', entity_type='user', entity_id=new_user.id, before_value=None, after_value contains user fields.

### TC-011-006: User update event is logged
**Requirement:** REQ-059
**Scenario:** Admin updates a user's full_name.
**Expected:** AuditLog entry created with action='update', entity_type='user', before_value contains old name, after_value contains new name.

### TC-011-007: User role change event is logged
**Requirement:** REQ-059
**Scenario:** Admin changes a user's role from 'viewer' to 'editor'.
**Expected:** AuditLog entry created with action='role_change', entity_type='user', before_value includes old role, after_value includes new role.

### TC-011-008: User archive event is logged
**Requirement:** REQ-059
**Scenario:** Admin archives a user.
**Expected:** AuditLog entry created with action='archive', entity_type='user', before_value has is_archived=False, after_value has is_archived=True.

### TC-011-009: Team create event is logged
**Requirement:** REQ-059
**Scenario:** create_team() is called.
**Expected:** AuditLog entry with action='create', entity_type='team'.

### TC-011-010: Team update event is logged
**Requirement:** REQ-059
**Scenario:** update_team() is called.
**Expected:** AuditLog entry with action='update', entity_type='team'.

### TC-011-011: Team archive event is logged
**Requirement:** REQ-059
**Scenario:** archive_team() is called.
**Expected:** AuditLog entry with action='archive', entity_type='team'.

### TC-011-012: Product create/update/archive events are logged
**Requirement:** REQ-059
**Scenario:** create_product(), update_product(), archive_product() are called in sequence.
**Expected:** Three AuditLog entries: action='create', action='update', action='archive' with entity_type='product'.

### TC-011-013: Repository create/update/archive events are logged
**Requirement:** REQ-059
**Scenario:** create_repository(), update_repository(), archive_repository() are called in sequence.
**Expected:** Three AuditLog entries: action='create', action='update', action='archive' with entity_type='repository'.

### TC-011-014: Template create/update/archive events are logged
**Requirement:** REQ-059
**Scenario:** create_template(), update_template(), archive_template() are called in sequence.
**Expected:** Three AuditLog entries with entity_type='template'.

### TC-011-015: Template artifact changes log template_change events
**Requirement:** REQ-059
**Scenario:** add_artifact(), update_artifact(), remove_artifact() are called.
**Expected:** Three AuditLog entries with action='template_change', entity_type='template'.

### TC-011-016: Audit log entries are never deleted (permanent preservation)
**Requirement:** REQ-048
**Scenario:** AuditLog entries exist; no delete operation is available on the model or service.
**Expected:** AuditLog model has no delete or archive methods. Entries persist indefinitely.

### TC-011-017: Admin can access audit log list page
**Requirement:** REQ-061
**Scenario:** Admin user requests GET /audit/.
**Expected:** Returns HTTP 200 with audit log HTML.

### TC-011-018: Non-admin cannot access audit log list page
**Requirement:** REQ-061
**Scenario:** Editor or Viewer user requests GET /audit/.
**Expected:** Returns HTTP 403 or redirect to login/forbidden.

### TC-011-019: Unauthenticated user cannot access audit log
**Requirement:** REQ-061
**Scenario:** Unauthenticated request to GET /audit/.
**Expected:** Redirects to /login.

### TC-011-020: Audit log list supports entity_type filter
**Requirement:** REQ-062
**Scenario:** Admin requests GET /audit/?entity_type=user with user and team entries present.
**Expected:** Only user entries returned in response.

### TC-011-021: Audit log list supports action filter
**Requirement:** REQ-062
**Scenario:** Admin requests GET /audit/?action=create.
**Expected:** Only create entries returned.

### TC-011-022: Audit log list supports text search
**Requirement:** REQ-062
**Scenario:** Admin requests GET /audit/?q=some_email with matching entries.
**Expected:** Returns 200 with matching entries.

### TC-011-023: Audit log list supports HTMX partial
**Requirement:** REQ-062
**Scenario:** HTMX request to GET /audit/ with HX-Request header.
**Expected:** Returns partial HTML without <html> or <!DOCTYPE.

### TC-011-024: Audit log list is in reverse chronological order
**Requirement:** REQ-062
**Scenario:** Multiple entries exist with different timestamps.
**Expected:** Entries returned newest-first.

### TC-011-025: Audit log list supports pagination
**Requirement:** REQ-062
**Scenario:** More than 50 entries exist; admin requests page 2.
**Expected:** Returns HTTP 200 with second page of results.
