## Test Cases

### TC-006-001: List templates — active only by default
**Requirement:** REQ-031
**Description:** GET /templates/ returns only active (non-archived) templates by default.
**Steps:** Log in as any user, GET /templates/. Verify archived templates are not shown.
**Expected:** Response 200. Only active templates listed.

### TC-006-002: List templates — show archived
**Requirement:** REQ-031
**Description:** GET /templates/?archived=1 includes archived templates.
**Steps:** Log in as any user, GET /templates/?archived=1.
**Expected:** Response 200. Archived templates appear in list.

### TC-006-003: Admin can create a template
**Requirement:** REQ-031
**Description:** Admin creates a new template via POST /templates/new.
**Steps:** Log in as Admin, POST /templates/new with name and description.
**Expected:** Template created, redirect to list, success flash message.

### TC-006-004: Non-admin cannot create a template
**Requirement:** REQ-031
**Description:** Editor and Viewer are forbidden from creating templates.
**Steps:** Log in as Viewer, GET /templates/new.
**Expected:** HTTP 403.

### TC-006-005: Admin can edit a template
**Requirement:** REQ-031
**Description:** Admin updates template name and description.
**Steps:** Log in as Admin, POST /templates/<id>/edit with new name.
**Expected:** Template updated, redirect to list, success flash.

### TC-006-006: Non-admin cannot edit a template
**Requirement:** REQ-031
**Description:** Viewer is forbidden from editing templates.
**Steps:** Log in as Viewer, GET /templates/<id>/edit.
**Expected:** HTTP 403.

### TC-006-007: Template name uniqueness enforced on create
**Requirement:** REQ-032
**Description:** Creating a template with a duplicate name is rejected.
**Steps:** Create template "My Template". Attempt to create another "My Template".
**Expected:** TemplateServiceError raised with message indicating duplicate name.

### TC-006-008: Template name uniqueness enforced on update
**Requirement:** REQ-032
**Description:** Renaming a template to match an existing template name is rejected.
**Steps:** Create "Alpha" and "Beta". Rename "Beta" to "Alpha".
**Expected:** TemplateServiceError raised with duplicate name message.

### TC-006-009: Archive template with no active repositories succeeds
**Requirement:** REQ-033, REQ-049
**Description:** A template not used by any active repository can be archived.
**Steps:** Create a template. Call archive_template().
**Expected:** Template is archived (is_archived=True, is_active=False).

### TC-006-010: Archive template blocked by active repositories
**Requirement:** REQ-033, REQ-049
**Description:** A template used by active repositories cannot be archived.
**Steps:** (Future, when Repository model exists) Assign template to active repo. Call archive_template().
**Expected:** TemplateServiceError raised listing blocking repository names.

### TC-006-011: Non-admin cannot archive a template
**Requirement:** REQ-031
**Description:** Viewer is forbidden from archiving templates.
**Steps:** Log in as Viewer, POST /templates/<id>/archive.
**Expected:** HTTP 403.

### TC-006-012: Archived template can be reactivated
**Requirement:** REQ-031
**Description:** Admin restores an archived template.
**Steps:** Archive a template. POST /templates/<id>/reactivate.
**Expected:** Template restored (is_archived=False, is_active=True).

### TC-006-013: Optimistic locking on template update
**Requirement:** REQ-031
**Description:** update_template with wrong version raises TemplateServiceError.
**Steps:** Fetch template version. Attempt update with version+999.
**Expected:** TemplateServiceError raised with concurrency message.

### TC-006-014: Unauthenticated access to template list redirects to login
**Requirement:** REQ-031
**Description:** Unauthenticated GET /templates/ redirects to login.
**Steps:** Without logging in, GET /templates/.
**Expected:** Redirect to /login.
