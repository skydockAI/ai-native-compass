# TS-008: Template Change Propagation & Repository Duplication

## Test Suite ID
TS-008

## Delivery Item
[DI-008](../delivery-items/DI-008.md)

## Requirements Covered
- REQ-042 ‚Äî Template Artifact Addition Propagation
- REQ-043 ‚Äî Template Artifact Rename Propagation
- REQ-044 ‚Äî Template Artifact Removal Behavior
- REQ-045 ‚Äî List Option Editing and Deactivation
- REQ-052 ‚Äî Repository Duplication

---

## Test Cases

### TC-008-001 ‚Äî Artifact addition propagates to existing active repos (REQ-042)

**Given** a template has two active repositories (R1, R2) and one archived repository (R3)
**When** an admin adds a new artifact to the template
**Then** empty `RepositoryArtifactValue` rows are created for R1 and R2
**And** no row is created for R3 (archived)

---

### TC-008-002 ‚Äî Artifact addition propagation uses null values (REQ-042)

**Given** a template with an active repository
**When** a new document artifact is added to the template
**Then** the propagated `RepositoryArtifactValue` row has all value columns as NULL

---

### TC-008-003 ‚Äî Artifact addition does not propagate to repos using other templates (REQ-042)

**Given** two templates T1 and T2, each with one active repository
**When** an artifact is added to T1
**Then** only the T1 repository receives a new `RepositoryArtifactValue` row
**And** the T2 repository is unchanged

---

### TC-008-004 ‚Äî Artifact removal soft-deletes the artifact (REQ-044)

**Given** a template with an active artifact that has stored values in a repository
**When** the artifact is removed
**Then** `TemplateArtifact.is_active` is set to `False`
**And** the `TemplateArtifact` row still exists in the database
**And** the `RepositoryArtifactValue` rows for that artifact still exist

---

### TC-008-005 ‚Äî Removed artifacts are hidden from repository UI (REQ-044)

**Given** a repository with artifact values
**When** one of the template's artifacts is soft-deleted
**Then** the removed artifact does not appear in the repository detail page
**And** the remaining active artifacts are still shown

---

### TC-008-006 ‚Äî Removed artifact not shown in new repository create form (REQ-044)

**Given** a template with one active and one removed (soft-deleted) artifact
**When** a user loads the create repository form and selects the template
**Then** only the active artifact is shown in the artifact fields partial

---

### TC-008-007 ‚Äî Artifact rename updates display everywhere (REQ-043)

**Given** a repository with an artifact value for artifact "Old Name"
**When** the artifact is renamed to "New Name"
**Then** the repository detail page shows "New Name" for that artifact
**And** the stored `RepositoryArtifactValue` row is unchanged (still linked by FK)

---

### TC-008-008 ‚Äî Re-adding artifact after removal creates new artifact (REQ-044)

**Given** a template with a removed (soft-deleted) artifact
**When** an admin adds a new artifact with the same name
**Then** a new `TemplateArtifact` row is created
**And** propagation creates new `RepositoryArtifactValue` rows for active repos

---

### TC-008-009 ‚Äî Duplicate repository copies all artifact values (REQ-052)

**Given** a repository with artifact values set
**When** the repository is duplicated with a new name and URL
**Then** the duplicate has identical `RepositoryArtifactValue` rows for all active artifacts

---

### TC-008-010 ‚Äî Duplicate repository copies all shared attribute values (REQ-052)

**Given** a repository with custom shared attribute values set
**When** the repository is duplicated
**Then** the duplicate has identical `RepositorySharedAttributeValue` rows

---

### TC-008-011 ‚Äî Duplicate repository requires unique URL (REQ-052)

**Given** a repository with a known URL
**When** a duplicate is attempted with the same URL
**Then** the service raises an error indicating the URL already exists

---

### TC-008-012 ‚Äî Duplicate repository does not copy product links (REQ-052)

**Given** a repository linked to products via `product_repository`
**When** the repository is duplicated
**Then** the duplicate has no product links

---

### TC-008-013 ‚Äî Duplicate uses same template as source (REQ-052)

**Given** a repository using template T
**When** the repository is duplicated
**Then** the duplicate has the same `template_id`

---

### TC-008-014 ‚Äî Duplicate route accessible to Admin and Editor (REQ-052)

**Given** an admin or editor user
**When** GET `/repositories/<id>/duplicate`
**Then** the duplicate form is displayed (HTTP 200)

---

### TC-008-015 ‚Äî Duplicate route not accessible to Viewer (REQ-052)

**Given** a viewer user
**When** GET `/repositories/<id>/duplicate`
**Then** HTTP 403 is returned

---

### TC-008-016 ‚Äî Duplicate form pre-fills name and blank URL (REQ-052)

**Given** a repository named "Alpha Repo"
**When** an admin loads the duplicate form
**Then** the name field is pre-filled with "Copy of Alpha Repo"
**And** the URL field is empty

---

### TC-008-017 ‚Äî Successful duplication via route redirects to new repo detail (REQ-052)

**Given** an editor is on the duplicate form
**When** a valid name and unique URL are submitted
**Then** HTTP redirect leads to the new repository's detail page

---

### TC-008-018 ‚Äî Duplicate of unknown repo returns redirect with error (REQ-052)

**Given** a repo ID that does not exist
**When** GET `/repositories/99999/duplicate`
**Then** the response redirects and shows "not found"

---

### TC-008-019 ‚Äî List option deactivation hides option from new selections (REQ-045)

**Given** a list artifact with an active option
**When** the option is deactivated
**Then** the option does not appear in the dropdown for new repository artifact values

---

### TC-008-020 ‚Äî Deactivated list option still shown on repos that selected it (REQ-045)

**Given** a repository with a list artifact value pointing to an option
**When** that option is deactivated
**Then** the repository detail page still displays the option's value
