# DI-009: Product Management & Repository Linking

## Delivery Item ID
DI-009

## Title
Product Management & Repository Linking

## Objective
Implement the Product entity with full CRUD, the many-to-many Product-Repository linking/unlinking capability, and archive protection rules for both Products and Repositories based on their link dependencies.

## Included Requirements
- **REQ-023** — Product Entity Definition
- **REQ-024** — Product Name Uniqueness
- **REQ-025** — Product-Repository Many-to-Many Relationship
- **REQ-026** — Product Archive Protection
- **REQ-053** — Product CRUD Operations
- **REQ-054** — Independent Product-Repository Linking
- **REQ-049** — Dependency-Based Archive Blocking (Product and Repository portions)

## Scope

### In Scope
- Product model (`app/models/product.py`):
  - Fields: `name` (unique), `description`, `is_active`, `is_archived`, `created_at`, `updated_at`, `version`
  - Database-level unique constraint on `name`
- Product service (`app/services/product_service.py`):
  - Create, read, update, archive operations
  - Name uniqueness validation
  - Archive protection: cannot archive a product linked to active repositories
  - Return list of blocking repositories when archive is denied
- Product routes (`app/routes/products.py`):
  - Product list (active/archived filter), detail, create, edit, archive
  - Admin and Editor access for create/edit/archive; Viewer read-only
- Product-Repository linking:
  - Linking and unlinking managed from the Repository edit page
  - Products and Repositories can be created independently and linked later
  - Linking is optional at creation time
  - UI supports managing links from both Product detail and Repository edit pages
  - HTMX partial for product link management (`product_links.html`)
- Repository archive protection update:
  - Repository cannot be archived while linked to active Products
  - Users must manually remove Product links before archiving
  - UI displays blocking Product dependencies
- Product Jinja2 templates:
  - Product list page with active/archived toggle
  - Product detail page showing linked repositories
  - Product create/edit form
- Alembic migration for Product table (product_repository association table already created in DI-007)

### Out of Scope
- Dashboard product counts (DI-010)
- Audit logging of product actions (DI-011)

## Key Implementation Notes
- Products and Repositories have a many-to-many relationship via the `product_repository` association table
- Repository data is stored once and shared across all linked Products (REQ-030 — already enforced by the data model)
- Linking/unlinking does not affect repository data or audit history
- Product archive blocking: a Product cannot be archived if linked to any active (non-archived) Repository
- Repository archive blocking (new in this DI): a Repository cannot be archived while it has active Product links — users must manually unlink first
- The UI should support managing links from both directions (Product detail shows linked repos; Repository edit shows linked products)
- Product fields are fixed in v1: Name and Description only
