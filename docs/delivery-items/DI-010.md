# DI-010: Dashboard, Filtering & UI Polish

## Delivery Item ID
DI-010

## Title
Dashboard, Filtering & UI Polish

## Objective
Implement the dashboard landing page with summary cards, add filtering capabilities to all list views, ensure all core UI pages are complete and navigable, and polish the overall UI for usability and a business-friendly appearance.

## Included Requirements
- **REQ-055** — Dashboard-First UI Design
- **REQ-056** — Core UI Pages
- **REQ-057** — Dashboard Summary Cards
- **REQ-058** — List Filtering Capabilities
- **REQ-067** — UI Usability

## Scope

### In Scope
- Dashboard page (`app/routes/dashboard.py`, `templates/dashboard/index.html`):
  - Default landing page after login
  - Summary cards showing counts of active: Products, Teams, Repositories, Templates, Users
  - Counts exclude archived entities
  - Business-friendly, data-visibility-first design
- Dashboard route and service:
  - Aggregation queries for entity counts
  - HTMX-driven card updates (optional enhancement)
- List filtering across all entity list pages:
  - Repositories: filter by Product, Team, Template, active/archived status
  - Products: filter by active/archived status
  - Teams: filter by active/archived status
  - Templates: filter by active/archived status
  - Users: filter by role, active/archived status
  - Filters can be combined
  - HTMX-driven filtering (partial page updates without full reload)
- Reusable filter bar component (`templates/components/filter_bar.html`)
- Core UI page completeness audit:
  - Login page (DI-002)
  - Dashboard (this DI)
  - Products list/detail (DI-009)
  - Repositories list/detail/create/edit (DI-007)
  - Templates list/detail (DI-005)
  - User management — Admin only (DI-003)
  - Team management (DI-004)
  - Audit log — Admin only (DI-011)
- UI usability polish:
  - Clean, modern, business-friendly appearance
  - Intuitive form layouts with clear labels and validation messages
  - Logical navigation flow (sidebar, breadcrumbs)
  - Clear error messages for constraint violations
  - Responsive layout for desktop browsers
  - Consistent styling across all pages

### Out of Scope
- Full-text search across all entities (deferred)
- CSV/Excel export (deferred)
- Advanced analytics (deferred)
- Audit log page (DI-011)

## Key Implementation Notes
- Dashboard is the default redirect target after login (not a generic landing page)
- Summary cards should use the reusable `summary_card.html` component from the architecture
- Filtering should use HTMX to update list content without full page reloads
- The filter bar component should be generic enough to work across all list pages with different filter options
- Repository list filtering by Product, Team, and Template requires joins across the data model
- All filtering is server-side; no client-side JavaScript filtering needed
- Ensure consistent pagination across all list pages
- Breadcrumbs should reflect the current navigation path
