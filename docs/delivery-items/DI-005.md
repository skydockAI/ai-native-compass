# DI-005: Repo Template & Artifact Management

## Delivery Item ID
DI-005

## Title
Repo Template & Artifact Management

## Objective
Implement the Repo Template entity with full artifact management — including the four artifact types (Document, Skill, Agent, Other), value type definitions, list option lifecycle, required flag for Other-type artifacts, and template archive protection.

- **REQ-033** — Template Archive Protection
- **REQ-037** — Template Artifact Structure
- **REQ-038** — Document/Skill/Agent Artifact Values
- **REQ-039** — Other Artifact Value Types
- **REQ-040** — Required Flag for Other Artifacts
- **REQ-041** — List Option Lifecycle Management
- **REQ-049** — Dependency-Based Archive Blocking (Template portion)

## Scope

### In Scope
- RepoTemplate model (`app/models/template.py`):
  - Fields: `name`, `description`, `is_active`, `is_archived`, `created_at`, `updated_at`, `version`
  - Database-level unique constraint on `name`
- TemplateArtifact model:
  - Fields: `template_id` (FK), `type` (enum: Document/Skill/Agent/Other), `name`, `description`, `value_type` (for Other: text/number/boolean/list), `is_required`, `display_order`, `is_active`, `created_at`, `updated_at`
- ArtifactListOption model:
  - Fields: `artifact_id` (FK), `value`, `display_order`, `is_active`, `created_at`
- Template service (`app/services/template_service.py`):
  - Template CRUD operations
  - Artifact CRUD (add, edit, remove artifacts within a template)
  - Name uniqueness validation
  - Archive protection: check for active repositories before allowing archive
  - List option management: add, edit, deactivate, delete (only if unused)
- Template routes (`app/routes/templates.py`):
  - Template list, detail, create, edit, archive (Admin-only)
  - Artifact management within template detail/edit pages
  - List option management for Other:list artifacts
- Template Jinja2 templates:
  - Template list page with active/archived toggle
  - Template detail page showing all artifacts
  - Template create/edit form with artifact management
  - Artifact form sections for each type
  - List option management UI
- Alembic migrations for RepoTemplate, TemplateArtifact, and ArtifactListOption tables

### Out of Scope
- Template change propagation to existing repositories (DI-008)
- Repository creation workflow using templates (DI-007)
- Audit logging of template actions (DI-011)

## Key Implementation Notes
- Only Admin users can manage templates and their artifacts
- Document/Skill/Agent artifacts have exactly three possible values: Yes, No, N/A (stored as text)
- Other-type artifacts support four value types: text, number, boolean (tri-state: True/False/N/A), list
- Value type is set at artifact creation and should not change after repositories use the artifact
- Required flag only applies to Other-type artifacts
- List options that are in use by any repository cannot be deleted but can be deactivated
- Deactivated list options are hidden from new selections but preserved on existing records
- Artifacts use `display_order` for ordering in the UI
- Artifact identity is tracked by internal ID, not by name (important for rename propagation later)
