# DI-006: Shared Repository Attributes

## Delivery Item ID
DI-006

## Title
Shared Repository Attributes

## Objective
Implement the shared attribute system that applies to all repositories regardless of their template — including the four protected default attributes (Name, Team, URL, Description) and the ability for Admins to define custom text-based shared attributes.

## Included Requirements
- **REQ-034** — Default Shared Repository Attributes
- **REQ-035** — Custom Shared Attributes
- **REQ-036** — Default Shared Attribute Protection

## Scope

### In Scope
- SharedAttributeDefinition model (`app/models/shared_attribute.py`):
  - Fields: `name` (unique), `is_default`, `is_active`, `created_at`, `updated_at`
  - Seed the four default attributes (Name, Team, URL, Description) with `is_default=True`
- RepositorySharedAttributeValue model:
  - Fields: `repository_id` (FK), `attribute_id` (FK), `value` (text)
  - Unique constraint on `(repository_id, attribute_id)`
- Shared attribute admin UI:
  - List of all shared attributes (defaults shown as protected/non-editable)
  - Admin can add new custom shared attributes (text-only)
  - Admin can edit or deactivate custom attributes
  - Default attributes cannot be removed or renamed
- Service layer for shared attribute management:
  - CRUD for custom attributes
  - Protection enforcement for default attributes
  - Seeding default attributes on application startup (if not already present)
- Alembic migrations for SharedAttributeDefinition and RepositorySharedAttributeValue tables

### Out of Scope
- Repository creation/editing forms that use shared attributes (DI-007)
- Filtering by shared attribute values (DI-010)
- Audit logging (DI-011)

## Key Implementation Notes
- The four default shared attributes map to native columns on the Repository model (Name, Team, URL, Description per the architecture). The SharedAttributeDefinition table tracks them for the admin UI, but their values come from Repository columns.
- Custom shared attributes use the EAV pattern: SharedAttributeDefinition + RepositorySharedAttributeValue
- Custom attributes are text-only in v1
- Default attributes are seeded on startup (similar to admin seeding) and marked with `is_default=True`
- The `is_default` flag protects default attributes from removal or renaming
- Only Admin users can manage shared attribute definitions
- Custom attributes are optional by default for all repositories
