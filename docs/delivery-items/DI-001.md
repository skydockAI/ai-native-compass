## Delivery Item ID
DI-001

## Title
Project Foundation & Infrastructure Setup

## Objective
Establish the complete project skeleton, development infrastructure, and deployment stack so that all subsequent delivery items can build on a working, runnable application with database connectivity, migrations, test framework, and base UI layout.

## Scope

### In Scope
- Flask application factory (`app/__init__.py`, `app/config.py`, `app/extensions.py`)
- Configuration loading from environment variables (`.env` / `.env.sample`)
- SQLAlchemy 2.x integration with Flask-SQLAlchemy
- Alembic / Flask-Migrate setup for database migrations
- Base model mixin establishing soft-delete pattern (`is_active`, `is_archived`, `created_at`, `updated_at`, `version` for optimistic locking)
- Docker Compose configuration with Flask web service and PostgreSQL 16 service
- Dockerfile with Python 3.12, Gunicorn WSGI server
- Host-mounted PostgreSQL data volume (`.data/postgres/`)
- `.gitignore` for `.data/`, `.env`, `__pycache__`, etc.
- Base Jinja2 layout template (`base.html`) with Bootstrap 5, sidebar navigation, and navbar
- Reusable UI components: pagination, flash messages, confirm modal partials
- HTMX 2.x inclusion (CDN or static)
- Flask-WTF integration for CSRF protection
- pytest + pytest-flask test infrastructure (`tests/conftest.py`, test database configuration)
- Project `requirements.txt` with all dependencies
- Application entrypoint that runs migrations on startup

### Out of Scope
- Authentication and login (DI-002)
- Any business entity models beyond the base mixin
- Business logic and service layer implementations

## Key Implementation Notes
- Follow the application structure defined in the architecture document (Section 3)
- The base model mixin should include `id`, `is_active`, `is_archived`, `created_at`, `updated_at`, and `version` fields as common columns
- Docker Compose should use PostgreSQL 16 Alpine with a healthcheck; the web service should depend on the DB being healthy
- The `.env.sample` file should document all required environment variables: `DATABASE_URL`, `FLASK_SECRET_KEY`, `DB_PASSWORD`, `ADMIN_SEEDS`
- Sidebar navigation should include placeholder links for all sections (Dashboard, Products, Repositories, Templates, Teams, Users, Audit) â€” pages will be implemented in later DIs
- Use Bootstrap 5 for all UI styling; no custom CSS framework
- Gunicorn with 2 workers is sufficient for the target ~20 users
