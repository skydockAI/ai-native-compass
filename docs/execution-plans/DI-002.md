# Execution Plan: DI-002 — Authentication & User Session Management

## Delivery Item

[DI-002](../delivery-items/DI-002.md)

## Objective

Implement a complete, SSO-ready authentication system: local email/password login, secure session management, admin seeding from environment variables, and session invalidation on user archive/deactivation.

## Architecture Alignment

- Auth logic encapsulated in `app/auth/` — `authenticate_user()` is the single swap point for future SSO (REQ-010)
- Authorization logic is separate (`app/authz/`) and unchanged by auth method
- Session validation via `before_request` hook (REQ-007)
- Admin seeding runs at `create_app()` startup after migrations (REQ-008)

## Implementation Steps

### 1. User model (`src/app/models/user.py`)
- Extend `BaseModel` + `UserMixin` (Flask-Login interface)
- Fields: `email` (unique, lowercase-normalised), `full_name`, `password_hash`, `role` (UserRole enum), `is_seeded`
- Methods: `set_password()` (enforces ≥8 chars, hashes with Werkzeug), `check_password()`, `normalize_email()`

### 2. Auth module (`src/app/auth/`)
- `__init__.py` — exports `authenticate_user` as the single entry point
- `local.py` — implements `authenticate_user(email, password)`: normalise email, query user, verify hash, check active/archived; returns `None` on any failure (generic, never reveals which field is wrong)
- `forms.py` — `LoginForm` (Flask-WTF): email + password fields with CSRF
- `routes.py` — `/login` (GET/POST) and `/logout` (GET, login-required) blueprint

### 3. Admin seeding (`src/app/utils/seed.py`)
- Read `ADMIN_SEEDS` env var (comma-separated `email:password` pairs)
- Skip gracefully if table does not exist (testing / first-migration scenarios)
- Idempotent: skip entries where user already exists
- Creates users with `role=ADMIN`, `is_seeded=True`, hashed password

### 4. Update `src/app/__init__.py`
- Replace placeholder `user_loader` with real `User` lookup
- Set `login_manager.login_view = 'auth.login'`
- Register `auth_bp` blueprint
- Add `before_request` hook: if authenticated user is inactive or archived → logout + redirect
- Call `seed_admins()` at startup (wrapped in try/except for safety)

### 5. Protect dashboard route
- Add `@login_required` to `GET /`

### 6. Login template (`src/app/templates/auth/login.html`)
- Standalone page (no sidebar — not extending `base.html`)
- Uses design system CSS (main.css)
- Email + password fields with Bootstrap Icons, CSRF token, flash messages

### 7. Database migration
- `flask db migrate -m "add user table"` to generate migration
- `flask db upgrade` applied via `entrypoint.sh`

### 8. Add `email-validator` to `requirements.txt`
- Required by WTForms `Email()` validator

### 9. Update existing tests
- Dashboard and UI-polish tests that access `/` unauthenticated → use new `auth_client` fixture
- Add `test_user` and `auth_client` fixtures to `conftest.py`

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Email normalised to lowercase at application layer | Avoids PostgreSQL `citext` extension dependency; DB unique index enforces uniqueness since all values are already lowercase |
| `authenticate_user()` returns `None` for ALL failure types | REQ-001: never reveal which field is incorrect |
| `before_request` re-checks `is_active`/`is_archived` from DB on every request | REQ-007: immediate denial without waiting for session expiry |
| Seeding wrapped in `try/except` | Allows `create_app('testing')` to succeed even when DB is not yet ready |
| Login page is standalone (no sidebar) | Authenticated UI chrome is not relevant / meaningful on the login screen |
