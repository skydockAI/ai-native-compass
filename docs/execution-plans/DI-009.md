# Execution Plan: DI-009 — Product Management & Repository Linking

## Delivery Item
[DI-009](../delivery-items/DI-009.md)

## Overview
This plan describes the implementation of the Product entity with full CRUD, Product-Repository many-to-many linking/unlinking, and archive protection for both Products and Repositories based on their link dependencies.

---

## Step 1 — Product Model

**File:** `src/app/models/product.py`

Create the `Product` model extending `BaseModel`:
- Fields: `name` (String(255), unique, not null), `description` (Text, nullable)
- Inherits: `id`, `is_active`, `is_archived`, `created_at`, `updated_at`, `version` from `BaseModel`
- Relationship: `products` → `product_repository` association table (many-to-many with `Repository`)

**File:** `src/app/models/repository.py` (update)
- Add `products` relationship on `Repository` using the `product_repository` association table

---

## Step 2 — Product Service

**File:** `src/app/services/product_service.py`

Functions:
- `get_products(include_archived=False)` — list products ordered by name
- `get_product_by_id(product_id)` — fetch single product
- `create_product(name, description=None)` — create with name uniqueness check
- `update_product(product_id, name, description, expected_version=None)` — update with optimistic locking
- `archive_product(product_id)` — blocks if linked to any active repository
- `reactivate_product(product_id)` — reactivate archived product
- `link_repository(product_id, repo_id)` — add product-repository link
- `unlink_repository(product_id, repo_id)` — remove product-repository link

`ProductServiceError` exception class for user-facing errors.

---

## Step 3 — Repository Service Update

**File:** `src/app/services/repository_service.py` (update)

Modify `archive_repository()`:
- Check if repo is linked to any active Products before archiving
- If blocked, raise `RepositoryServiceError` listing the blocking product names

## Step 5 — Product Routes

**File:** `src/app/routes/products.py`

Blueprint `products_bp` at `/products`:
- `GET /products/` — list with active/archived filter
- `GET /products/<id>` — detail with linked repositories
- `GET/POST /products/new` — create form (Admin + Editor)
- `GET/POST /products/<id>/edit` — edit form (Admin + Editor)
- `POST /products/<id>/archive` — archive (Admin + Editor)
- `POST /products/<id>/reactivate` — reactivate (Admin + Editor)
- `POST /products/<id>/link-repo` — HTMX: link repository (Admin + Editor)
- `POST /products/<id>/unlink-repo` — HTMX: unlink repository (Admin + Editor)
