# Execution Plan: DI-006 ‚Äî Shared Repository Attributes

## Delivery Item
[DI-006](../delivery-items/DI-006.md)

## Objective
Implement shared attribute definitions (four protected defaults + admin-managed custom attributes) and their value storage model for future repository use.

---

## Implementation Steps

### 1. Model ‚Äî `src/app/models/shared_attribute.py`
- `SharedAttributeDefinition`: minimal model (not extending BaseModel ‚Äî no version/archive), fields: `id`, `name` (unique), `is_default`, `is_active`, `created_at`, `updated_at`
- `RepositorySharedAttributeValue`: EAV value table, fields: `id`, `repository_id` (FK to repos, deferred), `attribute_id` (FK), `value` (text), unique constraint on `(repository_id, attribute_id)`

### 2. Seeding ‚Äî `src/app/utils/seed.py`
- Add `seed_default_shared_attributes()` to create Name, Team, URL, Description with `is_default=True` if not already present
- Call from `create_app()` wrapped in try/except (same pattern as `seed_admins`)

### 3. Service ‚Äî `src/app/services/shared_attribute_service.py`
- `get_attributes(include_inactive=False)` ‚Äî list all, ordered by is_default desc, then name
- `get_attribute_by_id(attr_id)` ‚Äî fetch by PK
- `create_attribute(name)` ‚Äî custom only, enforces uniqueness, sets `is_default=False`
- `update_attribute(attr_id, name)` ‚Äî raises if `is_default=True`
- `deactivate_attribute(attr_id)` ‚Äî raises if `is_default=True`
- `reactivate_attribute(attr_id)` ‚Äî allowed for custom only

### 4. Forms ‚Äî `src/app/routes/forms.py`
- `SharedAttributeCreateForm`: name field
- `SharedAttributeEditForm`: name field (used for rename of custom attrs)

### 5. Routes ‚Äî `src/app/routes/shared_attributes.py`
- Blueprint prefix `/shared-attributes`
- `GET /` ‚Äî list (admin only for management view, but read access for all authenticated)
- `GET/POST /new` ‚Äî create (admin only)
- `GET/POST /<id>/edit` ‚Äî edit (admin only, protected attrs get 403)
- `POST /<id>/deactivate` ‚Äî deactivate (admin only, protected attrs get error)
- `POST /<id>/reactivate` ‚Äî reactivate (admin only)

### 6. Templates ‚Äî `src/app/templates/shared_attributes/`
- `list.html` ‚Äî table of attributes, defaults show lock icon/protected badge, custom show edit/deactivate
- `form.html` ‚Äî shared create/edit form

### 7. App Factory ‚Äî `src/app/__init__.py`
- Register `shared_attributes_bp`
- Call `seed_default_shared_attributes()` on startup wrapped in try/except

### 8. Base Template ‚Äî `src/app/templates/base.html`
- Add "Shared Attributes" link under Administration sidebar section (admin only)

### 9. Migration ‚Äî `migrations/versions/e4f6a8c0d123_add_shared_attribute_tables.py`
- Revision ID: `e4f6a8c0d123`
- Down-revision: `d3f5a7b9c012` (templates migration)
- Creates `shared_attribute_definitions` and `repository_shared_attribute_values` tables

### 10. Tests
- `tests/test_models/test_shared_attribute.py` ‚Äî model instantiation
- `tests/test_services/test_shared_attribute_service.py` ‚Äî CRUD + protection enforcement
- `tests/test_routes/test_shared_attributes.py` ‚Äî route access control + CRUD flows
- `tests/test_utils/test_seed_shared_attributes.py` ‚Äî seeding idempotency
