# Execution Plan: DI-001 — Project Foundation & Infrastructure Setup

## Delivery Item
[DI-001](../delivery-items/DI-001.md)

## Objective
Establish the complete project skeleton, development infrastructure, and deployment stack so that all subsequent delivery items can build on a working, runnable application with database connectivity, migrations, test framework, and base UI layout.

## Execution Steps

### Step 1: Project Dependencies
Create `requirements.txt` with all required Python packages:
- Flask 3.x, Flask-SQLAlchemy, Flask-Migrate, Flask-WTF, Flask-Login
- SQLAlchemy 2.x, Alembic, psycopg2-binary
- Gunicorn, python-dotenv
- pytest, pytest-flask (dev dependencies)

### Step 2: Flask Application Factory
Create the core application structure under `src/app/`:
- `src/app/__init__.py` — App factory (`create_app()`) that loads config, initializes extensions, registers blueprints, runs migrations on startup
- `src/app/config.py` — Configuration classes loading from environment variables (`DATABASE_URL`, `FLASK_SECRET_KEY`)
- `src/app/extensions.py` — Extension instances (SQLAlchemy, Migrate, LoginManager, CSRFProtect)

### Step 3: Base Model Mixin
Create `src/app/models/__init__.py` with a `BaseModel` mixin providing:
- `id` (Integer, primary key)
- `is_active` (Boolean, default True)
- `is_archived` (Boolean, default False)
- `created_at` (DateTime, server default)
- `updated_at` (DateTime, auto-update)
- `version` (Integer, default 1, for optimistic locking)

### Step 4: Docker Configuration
- `Dockerfile` — Python 3.12-slim, install requirements, copy app, expose 5000, Gunicorn CMD
- `docker-compose.yml` — Web service (build from Dockerfile, depends on db) + PostgreSQL 16 Alpine with healthcheck
- Host-mounted volume: `.data/postgres/` → `/var/lib/postgresql/data`

### Step 5: Environment Configuration
- Update `.env.sample` with all required variables (already partially done)
- Create working `.env` for local development

### Step 6: Database Migrations
- Initialize Flask-Migrate / Alembic in the app factory
- Create initial migration for the base schema
- Configure entrypoint to run `flask db upgrade` on startup

### Step 7: Base UI Layout
- `src/app/templates/base.html` — Master layout with Bootstrap 5 (CDN), HTMX 2.x (CDN), sidebar navigation, navbar, flash messages area, content block
- Sidebar links: Dashboard, Products, Repositories, Templates, Teams, Users, Audit (placeholder pages)

### Step 8: Reusable UI Components
- `src/app/templates/components/pagination.html`
- `src/app/templates/components/flash_messages.html`
- `src/app/templates/components/confirm_modal.html`

### Step 9: Dashboard Placeholder Route
- `src/app/routes/dashboard.py` — Blueprint with a simple index route rendering a placeholder page
- Register blueprint in app factory

### Step 10: Health Check Endpoint
- Add a `/health` route to the app for container health validation

### Step 11: Test Infrastructure
- `tests/conftest.py` — pytest fixtures with test app, test client, test database (SQLite for fast unit tests)
- Test configuration using in-memory SQLite or test PostgreSQL

### Step 12: Automated Tests
- Test app factory creates app successfully
- Test health check endpoint returns 200
- Test base model mixin fields exist
- Test CSRF protection is active
- Test database connection and migration
- Test base template renders with expected elements

### Step 13: Documentation Updates
- Update `docs/user-guide.md` with setup and startup instructions
- Update `README.md` with project overview and quick start
- Create test suite `docs/test-suites/TS-001.md`
- Update `docs/test-suites-index.md`

## Validation Criteria
1. `pytest` passes all tests
2. `docker compose up` starts both services successfully
3. Health check endpoint returns 200
4. Base layout renders with Bootstrap 5 sidebar and navbar
5. HTMX is loaded on the page
6. PostgreSQL data persists in `.data/postgres/`
7. Alembic migrations run on startup
