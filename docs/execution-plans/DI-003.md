## Implementation Steps

### Step 1: Authorization Module (`app/authz/`)
- Create `app/authz/decorators.py` with:
  - `@role_required(*roles)` decorator that checks `current_user.role` against allowed roles
  - Returns 403 Forbidden for unauthorized access
  - Must be applied after `@login_required`
- Update `app/authz/__init__.py` to export decorators

### Step 2: User Service (`app/services/user_service.py`)
- Implement `UserService` class with:
  - `get_users(include_archived=False)` — list users with optional archived filter
  - `get_user_by_id(user_id)` — get single user
  - `create_user(email, full_name, password, role)` — create user with email uniqueness check
  - `update_user(user_id, data, version)` — update user with optimistic locking
  - `archive_user(user_id)` — archive user with seeded admin protection
  - `reactivate_user(user_id)` — reactivate archived user
  - `change_password(user_id, current_password, new_password)` — self-service password change
  - `admin_reset_password(user_id, new_password)` — admin password reset
  - `change_role(user_id, new_role)` — role change with seeded admin protection
- All mutations enforce seeded admin protection (cannot demote, archive, or delete seeded admins)
- Email uniqueness is case-insensitive, including archived users

### Step 3: User Forms (`app/routes/forms/`)
- Create WTF forms:
  - `UserCreateForm` — email, full_name, password, confirm_password, role
  - `UserEditForm` — email, full_name, role (no password)
  - `ChangePasswordForm` — current_password, new_password, confirm_password
  - `AdminResetPasswordForm` — new_password, confirm_password

### Step 4: User Routes (`app/routes/users.py`)
- Create `users_bp` Blueprint with routes:
  - `GET /users` — user list (admin only), with active/archived filter
  - `GET /users/new` — create user form (admin only)
  - `POST /users` — create user (admin only)
  - `GET /users/<id>/edit` — edit user form (admin only)
  - `POST /users/<id>/edit` — update user (admin only)
  - `POST /users/<id>/archive` — archive user (admin only)
  - `POST /users/<id>/reactivate` — reactivate user (admin only)
  - `GET /users/change-password` — self-service password change form (any authenticated user)
  - `POST /users/change-password` — process password change (any authenticated user)
  - `GET /users/<id>/reset-password` — admin password reset form (admin only)
  - `POST /users/<id>/reset-password` — process admin password reset (admin only)
- All admin routes protected with `@role_required('admin')`

### Step 5: User Templates
- `templates/users/list.html` — user list with active/archived toggle, action buttons
- `templates/users/form.html` — create/edit form with role selector
- `templates/users/change_password.html` — self-service password change
- `templates/users/reset_password.html` — admin password reset

### Step 6: Update Base Template
- Make sidebar role-aware: Administration section (Users, Audit) visible only to admin
- Display current user name and role in navbar
- Add "Change Password" link to user menu

### Step 7: Register Blueprint
- Register `users_bp` in `app/__init__.py`

### Step 8: Database Migration
