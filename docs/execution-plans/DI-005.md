# Execution Plan: DI-005 ‚Äî Repo Template & Artifact Management

## Delivery Item
[DI-005](../delivery-items/DI-005.md)

## Overview

This plan implements the RepoTemplate entity with full artifact management: three related database tables, a service layer, Admin-only routes, and Jinja2 UI.

---

## Step 1 ‚Äî Models (`src/app/models/template.py`)

Create three SQLAlchemy models extending `BaseModel` (where applicable):

### `ArtifactType` (Enum)
- Values: `document`, `skill`, `agent`, `other`

### `ArtifactValueType` (Enum)
- Values: `text`, `number`, `boolean`, `list`
- Applies only to `Other`-type artifacts

### `RepoTemplate(BaseModel)`
- `name`: String(255), nullable=False, unique=True
- `description`: Text, nullable=True
- Inherits: `id`, `is_active`, `is_archived`, `created_at`, `updated_at`, `version`

### `TemplateArtifact(BaseModel)`
- `template_id`: Integer FK ‚Üí `repo_templates.id`
- `type`: Enum(`ArtifactType`), nullable=False
- `name`: String(255), nullable=False
- `description`: Text, nullable=True
- `value_type`: Enum(`ArtifactValueType`), nullable=True (only set for Other type)
- `is_required`: Boolean, default=False (Other-type only)
- `display_order`: Integer, default=0

### `ArtifactListOption`
- Does NOT extend BaseModel (no soft-delete needed at base level; uses its own `is_active`)
- `id`: Integer PK
- `artifact_id`: Integer FK ‚Üí `template_artifacts.id`
- `value`: String(255), nullable=False
- `display_order`: Integer, default=0
- `is_active`: Boolean, default=True
- `created_at`: DateTime, server_default=now()

## Step 2 ‚Äî Service (`src/app/services/template_service.py`)

Functions:
- `get_templates(include_archived=False)` ‚Üí list
- `get_template_by_id(template_id)` ‚Üí RepoTemplate | None
- `create_template(name, description)` ‚Üí RepoTemplate
- `update_template(template_id, name, description, expected_version)` ‚Üí RepoTemplate
- `archive_template(template_id)` ‚Üí RepoTemplate
- `reactivate_template(template_id)` ‚Üí RepoTemplate
- `add_artifact(template_id, type, name, description, value_type, is_required, display_order)` ‚Üí TemplateArtifact
- `update_artifact(artifact_id, name, description, is_required, display_order)` ‚Üí TemplateArtifact
- `remove_artifact(artifact_id)` ‚Äî hard delete (artifact removed from template)
- `add_list_option(artifact_id, value, display_order)` ‚Üí ArtifactListOption
- `update_list_option(option_id, value, display_order)` ‚Üí ArtifactListOption
- `deactivate_list_option(option_id)` ‚Üí ArtifactListOption
- `delete_list_option(option_id)` ‚Äî hard delete (only if unused)

Archive protection: check for active repositories using this template before allowing archive. Until DI-007, stub returns empty list.

## Step 4 ‚Äî Routes (`src/app/routes/templates.py`)

Blueprint prefix: `/templates`

| Route | Method | Description |
|-------|--------|-------------|
| `/` | GET | Template list (all authenticated users) |
| `/new` | GET, POST | Create template (Admin only) |
| `/<id>` | GET | Template detail with artifact list (all authenticated users) |
| `/<id>/edit` | GET, POST | Edit template (Admin only) |
| `/<id>/archive` | POST | Archive template (Admin only) |
| `/<id>/reactivate` | POST | Reactivate template (Admin only) |
| `/<id>/artifacts/new` | GET, POST | Add artifact (Admin only) |
| `/<id>/artifacts/<aid>/edit` | GET, POST | Edit artifact (Admin only) |
| `/<id>/artifacts/<aid>/remove` | POST | Remove artifact (Admin only) |
| `/<id>/artifacts/<aid>/options/new` | GET, POST | Add list option (Admin only) |
| `/<id>/artifacts/<aid>/options/<oid>/edit` | GET, POST | Edit list option (Admin only) |
| `/<id>/artifacts/<aid>/options/<oid>/deactivate` | POST | Deactivate list option (Admin only) |
| `/<id>/artifacts/<aid>/options/<oid>/delete` | POST | Delete list option if unused (Admin only) |
