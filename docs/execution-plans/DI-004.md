## Delivery Item
[DI-004](../delivery-items/DI-004.md)

## Requirements Covered
- REQ-019 — Team Entity Definition
- REQ-020 — Team Name Uniqueness
- REQ-022 — Team Archive Protection
- REQ-049 — Dependency-Based Archive Blocking (Team portion)

---

## Implementation Steps

### 1. Team Model (`src/app/models/team.py`)
- Extend `BaseModel` (provides `id`, `is_active`, `is_archived`, `created_at`, `updated_at`, `version`)
- Fields: `name` (String 255, nullable=False, unique=True), `description` (Text, nullable=True)
- Database-level unique constraint on `name` (REQ-020)

### 2. Alembic Migration (`migrations/versions/`)
- Create table `teams` with all columns and unique constraint on `name`
- Revision chains from the user migration

### 3. Team Service (`src/app/services/team_service.py`)
- `get_teams(include_archived)` — list teams
- `get_team_by_id(team_id)` — fetch by PK
- `create_team(name, description)` — validate name uniqueness, commit
- `update_team(team_id, name, description, expected_version)` — optimistic lock, uniqueness check
- `archive_team(team_id)` — check for blocking active repositories (returns list); returns `(team, blocking_repos)`
- `reactivate_team(team_id)` — restore archived team

Archive blocking note: Until DI-007 (Repository entity) is implemented, `archive_team` will always return an empty blocking list and proceed. The service is structured to accept the repository check once the Repository model is available.

### 4. Team Forms (`src/app/routes/forms.py`)
- `TeamCreateForm`: `name`, `description` (optional), `submit`
- `TeamEditForm`: `name`, `description`, `version` (hidden), `submit`

### 5. Team Routes (`src/app/routes/teams.py`)
- `GET /teams/` — list (active/archived filter); accessible by all authenticated users
- `GET /teams/new`, `POST /teams/new` — create; Admin + Editor only
- `GET /teams/<id>/edit`, `POST /teams/<id>/edit` — edit; Admin + Editor only
- `POST /teams/<id>/archive` — archive; Admin + Editor only
- `POST /teams/<id>/reactivate` — reactivate; Admin + Editor only

### 6. Templates
- `src/app/templates/teams/list.html` — table with active/archived toggle, action buttons
- `src/app/templates/teams/form.html` — create/edit form

### 7. App Factory Update
- Register `teams_bp` blueprint in `src/app/__init__.py`

### 8. Automated Tests
- `tests/test_services/test_team_service.py` — CRUD, uniqueness, archive protection
- `tests/test_routes/test_teams.py` — route access, RBAC, form submission

### 9. Documentation
- Update `docs/user-guide.md` with Teams section
- Update `docs/test-suites-index.md` with TS-005
