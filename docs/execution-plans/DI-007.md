# Execution Plan: DI-007 — Repository Core Management

## Delivery Item
[DI-007](../delivery-items/DI-007.md)

## Overview

Implement the Repository entity with the template-first creation workflow, editing rules, team assignment, URL uniqueness, immutable template assignment, shared attribute value entry, and template-derived artifact value management.

---

## Implementation Steps

### 1. Data Model

**File:** `src/app/models/repository.py`

Create the following models:

#### Repository (extends BaseModel)
- `name` — String(255), required
- `url` — String(2048), required, unique (database-level constraint)
- `description` — Text, optional
- `team_id` — FK → `teams.id`, required, indexed
- `template_id` — FK → `repo_templates.id`, required, indexed
- Relationships: `team`, `template`, `artifact_values`, `shared_attr_values`

#### RepositoryArtifactValue
- `repository_id` — FK → `repositories.id`, required, indexed
- `template_artifact_id` — FK → `template_artifacts.id`, required, indexed
- `value_text` — Text, nullable
- `value_number` — Numeric(18,6), nullable
- `value_boolean` — Boolean, nullable (tri-state: True/False/NULL)
- `value_list_option_id` — FK → `artifact_list_options.id`, nullable
- Unique constraint on `(repository_id, template_artifact_id)`

#### product_repository (association table)
- `product_id` — FK → `products.id`
- `repository_id` — FK → `repositories.id`
- Unique composite index on `(product_id, repository_id)`

Also add FK on `repository_shared_attribute_values.repository_id → repositories.id` via migration.

---

### 2. Service Layer

**File:** `src/app/services/repository_service.py`

Functions:
- `get_repositories(include_archived=False)` — list, ordered by name
- `get_repository_by_id(repo_id)` — single lookup or None
- `create_repository(name, url, description, team_id, template_id, artifact_values, shared_attr_values)`:
  - Validate name, url, team_id, template_id required
  - URL uniqueness (including archived)
  - Team must exist and be active
  - Template must exist and be active
  - Insert Repository row
  - Insert RepositoryArtifactValue rows for all active artifacts in the template
  - Insert/upsert RepositorySharedAttributeValue rows for all active custom attributes
  - Validate required artifact values on save
- `update_repository(repo_id, name, url, description, team_id, artifact_values, shared_attr_values, expected_version)`:
  - Optimistic locking (version check)
  - URL uniqueness (excluding current repo)
  - template_id immutable — silently ignored if passed
  - Team must be active
  - Validate required artifact values
  - Update artifact values (upsert)
  - Update shared attr values (upsert)
- `archive_repository(repo_id)` — soft-delete
- `reactivate_repository(repo_id)` — restore
- `get_artifact_values_map(repo_id)` — dict: artifact_id → RepositoryArtifactValue
- `get_shared_attr_values_map(repo_id)` — dict: attribute_id → RepositorySharedAttributeValue

### 4. Routes

**File:** `src/app/routes/repositories.py`

Blueprint: `repositories_bp`, url_prefix `/repositories`

| Method | URL | Description |
|--------|-----|-------------|
| GET | `/repositories/` | List (active/archived filter) |
| GET | `/repositories/<id>` | Detail page |
| GET/POST | `/repositories/new` | Create (step 1: template select) |
| GET | `/repositories/artifact-fields` | HTMX partial — load artifact fields for template |
| GET/POST | `/repositories/<id>/edit` | Edit repository |
| POST | `/repositories/<id>/archive` | Archive |
| POST | `/repositories/<id>/reactivate` | Reactivate |

Access:
- List/Detail: all authenticated users
- Create/Edit/Archive/Reactivate: Admin + Editor only
- HTMX endpoint: Admin + Editor only (called during create)
