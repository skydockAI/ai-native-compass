# REQ-004: Session-Based Authentication

## Priority
High

## Description
The system must use session-based authentication as supported by Flask. Sessions track the authenticated user and their role.

## Acceptance Criteria
- A server-side session is created upon successful login
- The session stores the user's identity and role
- Session is invalidated on logout
- Session follows standard Flask security practices (secure cookies, CSRF protection)
