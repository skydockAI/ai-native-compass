# REQ-010: SSO-Ready Authentication Design

## Priority
Medium

## Description
The authentication layer must be designed so that local authentication can later be replaced or extended with SSO. The architecture should decouple authentication logic from authorization logic.

## Acceptance Criteria
- Authentication logic is encapsulated in a module/layer that can be swapped or extended
- Authorization (role checks, permission enforcement) is separate from authentication
- The role model and authorization logic remain unchanged regardless of authentication method
- Future SSO integration can match authenticated email with an active user in the database
