# REQ-040: Required Flag for Other Artifacts

## Priority
Medium

## Description
Admin can mark an `Other`-type artifact as required. When an artifact is required, Repositories must provide a value for it.

## Acceptance Criteria
- Admin can set a "required" flag on Other-type artifacts during Template creation or editing
- Required artifacts must have a non-empty value when saving a Repository
- The UI visually indicates which artifacts are required
- Validation errors are shown if required artifact values are missing on save
- Document/Skill/Agent artifact types do not have a required flag (they always allow Yes/No/N/A)
