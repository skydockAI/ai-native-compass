# REQ-008: Admin User Seeding from Environment

## Priority
High

## Description
On application startup, the system reads a list of Admin email addresses from the `.env` file. For each configured email, if the Admin user does not exist in the database, the system creates the Admin user automatically.

## Acceptance Criteria
- `.env` contains a configuration key `ADMIN_SEEDS` with comma-separated `email:password` pairs (e.g., `admin@example.com:secretpass,admin2@example.com:otherpass`)
- On each application startup, the system checks for each configured Admin email
- If an Admin user does not exist, the system creates the user with the Admin role and the provided password (hashed)
- The initial password must meet the minimum length requirement of 8 characters
- Seeded Admin users are active and can log in immediately
- If the Admin already exists, no changes are made
- A `.env.sample` file is provided as a reference template for required environment variables

## Resolved Ambiguities
- Initial password for seeded admins is provided alongside the email in the `.env` file as `email:password` pairs.
- Full name for seeded admins defaults to the email prefix (the portion before `@`).
