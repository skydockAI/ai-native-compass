# REQ-002: Active User Login Restriction

## Priority
High

## Description
Only users with an active status may log in. Archived or inactive users must be denied access.

## Acceptance Criteria
- Login is rejected for users whose active status is false
- Login is rejected for users whose archived status is true
- An appropriate error message is displayed for inactive/archived users attempting to log in
- If a user is archived or deactivated while logged in, access must be denied on the next request
