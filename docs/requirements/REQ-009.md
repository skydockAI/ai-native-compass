# REQ-009: Seeded Admin Protection

## Priority
High

## Description
Admin users created from `.env` seeding cannot be demoted to a lower role or deleted/archived by another Admin through the UI or API.

## Acceptance Criteria
- Seeded Admin users are flagged or identified as environment-seeded
- The system prevents role changes for seeded Admin users
- The system prevents archiving or deletion of seeded Admin users
- Attempting to demote or delete a seeded Admin displays an appropriate error message
- Non-seeded Admin users can be managed normally
