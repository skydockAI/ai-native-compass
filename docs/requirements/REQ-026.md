# REQ-026: Product Archive Protection

## Priority
High

## Description
A Product cannot be archived if it is linked to any active (non-archived) Repository.

## Acceptance Criteria
- The system prevents archiving a Product that is linked to one or more active Repositories
- When archive is blocked, the UI displays a clear message explaining which active Repositories are linked
- Once all linked Repositories are archived or unlinked, the Product can be archived

## Ambiguities
- The raw requirements contain a conflict: Section 5.3 says "Product can not be archived if linked to an active Repository" but Section 8.2 says "Product may be archived without archiving or removing linked active Repositories." This requirement follows Section 5.3 (blocking archive) as the stricter, safer interpretation. This conflict should be resolved with stakeholders.
