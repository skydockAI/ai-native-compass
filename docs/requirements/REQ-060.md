# REQ-060: Audit Log Entry Structure

## Priority
High

## Description
Each audit log entry must store a defined set of data fields to provide a complete record of the action.

## Acceptance Criteria
- Each audit log entry includes:
  - Timestamp (datetime, auto-set)
  - User (who performed the action)
  - Entity type (e.g., Product, Repository, User, Team, Template)
  - Entity identifier (ID or name of the affected entity)
  - Action (e.g., Create, Update, Archive, Login, Logout)
  - Before value (state before the change, where applicable)
  - After value (state after the change, where applicable)
- Before/After values capture meaningful field-level changes, not entire entity snapshots
