# REQ-001: Local Email/Password Authentication

## Priority
High

## Description
The system must support local authentication using email and password. Users log in by providing their registered email address and password. Only users whose accounts already exist in the database may log in.

## Acceptance Criteria
- User can log in with a valid email and password combination
- Login is rejected if the email does not exist in the database
- Login is rejected if the password is incorrect
- An appropriate error message is shown on failed login (without revealing which field is incorrect)
- Successful login creates a session and redirects the user to the dashboard
