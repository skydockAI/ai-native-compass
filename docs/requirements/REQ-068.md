# REQ-068: Security Best Practices

## Priority
High

## Description
The v1 application must follow standard security practices for password storage, session handling, and authorization enforcement.

## Acceptance Criteria
- Passwords are stored using strong hashing (e.g., bcrypt, argon2)
- Session handling follows standard Flask security practices (secure cookies, HTTP-only, CSRF protection)
- Authorization is enforced server-side on every request (not just UI-level hiding)
- Authentication can later be replaced or extended to SSO without changing authorization logic
