# REQ-023: Product Entity Definition

## Priority
High

## Description
A Product is a business-level entity that can be linked to one or more Repositories.

## Acceptance Criteria
- Product entity includes the following fields:
  - Name (string, required)
  - Description (string, optional)
  - Active status (boolean, required)
  - Created timestamp (datetime, auto-set)
  - Updated timestamp (datetime, auto-updated)
  - Archived status (boolean, required, default false)
- Products can be created, edited, and archived by Admin and Editor users
- Product fields are fixed in v1 (Name and Description only)
