# REQ-027: Repository Entity Definition

## Priority
High

## Description
A Repository is a global shared entity representing a code repository tracked for AI Native adoption.

## Acceptance Criteria
- Repository entity includes the following fields:
  - Name (string, required)
  - Team (foreign key to Team, required)
  - URL (string, required)
  - Description (string, optional)
  - Template (foreign key to Repo Template, required)
  - Additional shared attributes (dynamic, based on Admin configuration)
  - Template-derived values (dynamic, based on assigned Template's artifacts)
  - Active status (boolean, required)
  - Created timestamp (datetime, auto-set)
  - Updated timestamp (datetime, auto-updated)
  - Archived status (boolean, required, default false)
- Repositories can be created, edited, and archived by Admin and Editor users
