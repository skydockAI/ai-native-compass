# REQ-049: Dependency-Based Archive Blocking

## Priority
High

## Description
Archiving may be blocked when active dependencies exist. The UI must clearly show the dependency reason when archive is blocked.

## Acceptance Criteria
- Team cannot be archived if used by an active Repository
- Template cannot be archived if used by an active Repository
- Product cannot be archived if linked to an active Repository (see REQ-026 ambiguity note)
- Repository cannot be archived while linked to active Products; users must manually remove Product-Repository links before archiving
- The UI displays blocking dependencies but does not auto-remove them
- User may be archived even if audit history exists
- When archive is blocked, the UI displays a clear message listing the dependent entities
- The blocking check is enforced server-side
