# REQ-050: Repository Creation Workflow

## Priority
High

## Description
Repository creation follows a template-first workflow. The user selects a Template first, then the system displays shared fields and template artifacts for completion.

## Acceptance Criteria
- User opens the create Repository page
- User selects a Template first (from active Templates only)
- After Template selection, the system displays:
  - Default shared fields (Name, Team, URL, Description)
  - Custom shared attributes (if any)
  - Template-specific artifacts with appropriate input controls
- User completes required fields and saves
- System validates:
  - Repository URL is unique
  - Team is selected
  - Template is selected
  - Required artifact values are provided
- On successful validation, the Repository is created
