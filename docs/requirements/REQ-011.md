# REQ-011: Role Model - Three Roles

## Priority
High

## Description
The system supports exactly three user roles: Admin, Editor, and Viewer. Each user is assigned exactly one role.

## Acceptance Criteria
- The system defines exactly three roles: Admin, Editor, Viewer
- Every user in the system has exactly one of these roles
- Role assignment is stored in the user record
- Role values are enforced at the database level (e.g., enum or check constraint)
