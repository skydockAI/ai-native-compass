# REQ-012: Single Role Per User

## Priority
High

## Description
Each user must have exactly one role assigned. Multiple roles per user are not supported.

## Acceptance Criteria
- The user data model enforces a single role field (not a list/set of roles)
- Role assignment is mandatory - a user cannot exist without a role
- Changing a user's role replaces the previous role entirely
