# REQ-019: Team Entity Definition

## Priority
High

## Description
A Team is a managed entity used to categorize Repositories. Each Repository belongs to exactly one Team.

## Acceptance Criteria
- Team entity includes the following fields:
  - Name (string, required)
  - Description (string, optional)
  - Active status (boolean, required)
  - Created timestamp (datetime, auto-set)
  - Updated timestamp (datetime, auto-updated)
  - Archived status (boolean, required, default false)
- Teams can be created, edited, and archived by Admin and Editor users
