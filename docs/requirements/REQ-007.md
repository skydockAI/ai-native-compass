# REQ-007: Immediate Access Denial for Archived/Removed Users

## Priority
High

## Description
If a user is removed or archived, access must be denied on the very next request. The system must not wait for a session to expire naturally.

## Acceptance Criteria
- When a user is archived or deactivated, any existing session for that user becomes invalid
- The next HTTP request from the archived/deactivated user is denied with an appropriate error
- The user is redirected to the login page after denial
