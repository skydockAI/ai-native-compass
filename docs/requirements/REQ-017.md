# REQ-017: User Email Uniqueness

## Priority
High

## Description
User email addresses must be unique across the entire system. No two users (active or archived) can share the same email.

## Acceptance Criteria
- A unique constraint is enforced on the email field at the database level
- Creating a user with a duplicate email is rejected with a clear error message
- The uniqueness check is case-insensitive

## Assumptions
- The raw requirements state "Email must be unique" but do not specify case sensitivity. This requirement assumes case-insensitive uniqueness (e.g., `User@Example.com` and `user@example.com` are considered the same).
