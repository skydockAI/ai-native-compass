# REQ-039: Other Artifact Value Types

## Priority
High

## Description
Artifacts of type `Other` support four value types: `text`, `number`, `boolean`, and `list`. Each type has specific behavior.

## Acceptance Criteria
- **Text**: Supports multiline text input
- **Number**: Numeric input field; no min/max validation required in v1
- **Boolean**: Allowed values are `True`, `False`, or `N/A`
- **List**: Single-select dropdown with Admin-defined options
- The value type is defined when the artifact is created in the Template
- Editors and Admins can update these values on Repository edit pages
