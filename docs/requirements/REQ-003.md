# REQ-003: Secure Password Storage

## Priority
High

## Description
User passwords must be hashed and stored securely. Plain-text passwords must never be stored in the database.

## Acceptance Criteria
- Passwords are hashed using a strong hashing algorithm (e.g., bcrypt, argon2) before storage
- Plain-text passwords are never persisted to the database
- Password comparison is done via hash verification, not plain-text comparison
- Minimum password length of 8 characters is enforced on all password operations (creation, self-service change, admin reset)
