# REQ-028: Repository URL Uniqueness

## Priority
High

## Description
Repository URLs must be unique across the entire system. No two Repositories can share the same URL.

## Acceptance Criteria
- A unique constraint is enforced on the Repository URL field at the database level
- Creating or editing a Repository with a duplicate URL is rejected with a clear error message
- The uniqueness check applies to both active and archived Repositories
