# REQ-022: Team Archive Protection

## Priority
High

## Description
A Team cannot be archived if it is currently used by any active (non-archived) Repository.

## Acceptance Criteria
- The system prevents archiving a Team that is assigned to one or more active Repositories
- When archive is blocked, the UI displays a clear message explaining which active Repositories depend on the Team
- Once all dependent Repositories are archived or reassigned, the Team can be archived
