# REQ-016: User Entity Definition

## Priority
High

## Description
A User represents a person allowed to access the system. The User entity has a defined set of fields.

## Acceptance Criteria
- User entity includes the following fields:
  - Email (string, required)
  - Full name (string, required)
  - Role (enum: Admin/Editor/Viewer, required)
  - Active status (boolean, required)
  - Created timestamp (datetime, auto-set)
  - Updated timestamp (datetime, auto-updated)
  - Archived status (boolean, required, default false)
- User must be created by an Admin before login is possible
- Archived users cannot log in
