# REQ-025: Product-Repository Many-to-Many Relationship

## Priority
High

## Description
Products and Repositories have a many-to-many relationship. One Product can link to many Repositories, and one Repository can link to many Products. Repository adoption data is stored once and shared across all linked Products.

## Acceptance Criteria
- A Product can be linked to zero or more Repositories
- A Repository can be linked to zero or more Products
- Repository adoption/template data is stored once at the Repository level, not per Product link
- Linking or unlinking does not affect the Repository's data
- Removing a Repository-Product link removes only the relationship, not audit history
- UI provides a way to manage Product-Repository links from both sides
